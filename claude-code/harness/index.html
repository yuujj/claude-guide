하네스 엔지니어링 레퍼런스

LLM을 감싸는 스캐폴딩을 하네스(harness)라고 합니다. 같은 모델도 하네스 설정에 따라 생산성이 크게 달라집니다. 한 번 세팅해두면, 같은 설명을 반복하지 않아도 됩니다.

0/6 완료
참조

전체 settings.json 레퍼런스

모든 하네스 설정은 .claude/settings.json 한 파일에서 관리합니다.

권한 허용: permissions.allow — MCP 연결: mcpServers — 자동 실행: hooks

.claude/settings.json 
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Edit", "Write"],
    "deny":  ["Bash(rm -rf *)"]
  },
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  },
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "echo '[훅] 명령 실행 전'" }]
      }
    ],
    "Stop": [
      {
        "hooks": [{ "type": "command", "command": "echo '세션 종료'" }]
      }
    ]
  }
}

슬래시 커맨드는 settings.json 설정 없이 .claude/commands/*.md 파일만 만들면 자동 등록됩니다.

세팅 1

CLAUDE.md — 프로젝트 규칙 자동 주입

쉬움

프로젝트 루트에 CLAUDE.md 파일을 만들면, 클로드 코드가 매 세션 시작 시 자동으로 읽습니다. "이 프로젝트는 Python 3.11, pytest 사용. 커밋 메시지는 한국어로" 같은 규칙을 한 번만 적어두면 됩니다.

개발자용 예시

CLAUDE.md (개발자) 
# 프로젝트 규칙

## 기술 스택
- Python 3.11, FastAPI, PostgreSQL
- 테스트: pytest, coverage 90% 이상 유지

## 커밋 규칙
- 메시지는 한국어로, 동사로 시작 ("기능 추가", "버그 수정")
- PR 설명에는 변경 이유를 반드시 포함

## 금지 사항
- main 브랜치 직접 push 금지
- .env 파일 수정 금지

비개발자용 스타터 템플릿

CLAUDE.md (비개발자) 
# 내 작업 규칙

## 파일 처리
- 원본 삭제 전 backup/ 에 복사본 보관
- 파일명 날짜는 YYYY-MM-DD 형식
- 완료 후 변경 목록을 result.txt로 저장

## 금지 사항
- /Documents 외 경로 접근 금지
- 시스템 파일(.app, .dmg, .exe) 수정 금지

## 응답 방식
- 작업 전 한 줄 요약 후 진행
- 실수 가능성 있으면 먼저 확인 요청

시작 시 현재 작업 디렉토리부터 git 루트까지 올라가며 CLAUDE.md를 모두 읽습니다. 하위 폴더의 CLAUDE.md는 해당 폴더의 파일에 접근할 때 자동으로 추가 로드됩니다. 모노레포에서 서비스별로 다른 규칙을 적용할 때 유용합니다.

세팅 2

슬래시 커맨드 — 자주 쓰는 작업 한 줄로

보통

.claude/commands/ 폴더에 마크다운 파일을 만들면 슬래시 커맨드로 자동 등록됩니다. 같은 프롬프트를 두 번 이상 쓰고 있다면 커맨드로 만들 때입니다.

.claude/commands/weekly-report.md 
data/ 폴더의 이번 주 CSV를 읽고,
sales-template.xlsx 양식에 맞춰 수치를 채운 뒤,
reports/YYYY-MM-DD_weekly.pdf 로 저장해줘.

이 파일을 만들면 채팅창에서 /weekly-report만 입력하면 됩니다.

$ARGUMENTS 플레이스홀더를 사용하면 됩니다. 예: /deploy production처럼 쓰면 커맨드 파일의 $ARGUMENTS 자리에 "production"이 들어갑니다.

.claude/commands/deploy.md
$ARGUMENTS 환경에 배포해줘.
먼저 테스트를 실행하고 통과 시에만 진행해줘.
세팅 3

Hooks — 특정 이벤트에 자동 실행

고급

settings.json의 hooks 배열에 셸 명령을 등록하면, 커밋 전 자동 린트·테스트, 파일 저장 후 포맷터 실행, 세션 종료 시 요약 생성 등을 자동화할 수 있습니다.

이벤트 종류

이벤트실행 시점활용 예
PreToolUse도구 실행 직전위험한 명령 전 확인, 로그 기록
PostToolUse도구 실행 직후파일 저장 후 포맷터 실행
Stop세션 종료변경 내역 요약 저장

실전 예시 — Bash 실행 전 확인 + 파일 저장 후 prettier

.claude/settings.json 
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "echo '[훅] Bash 실행 전 검사 중'" }]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "npx prettier --write . 2>/dev/null || true" }]
      }
    ],
    "Stop": [
      {
        "hooks": [{ "type": "command", "command": "echo '세션 종료'" }]
      }
    ]
  }
}

훅 명령은 일반 셸 명령이라면 무엇이든 사용할 수 있습니다. CLAUDE_PROJECT_DIR 환경 변수로 현재 프로젝트 루트 경로를 참조할 수 있습니다. 도구별 입력값은 stdin으로 JSON 형태로 전달됩니다.

세팅 4

권한 모드 — 안전하게 자율성 늘리기

고급

settings.json의 permissions.allow로 매번 승인하지 않아도 되는 도구를 지정하고, permissions.deny로 특정 명령을 차단합니다. 읽기 전용 → 파일 편집 → 명령 실행 순으로 단계적으로 확대하는 것을 권장합니다.

단계별 권한 확대

단계permissions.allow설명
1단계 · 탐색 ["Read", "Glob", "Grep"] 파일 읽기만 가능 — 수정 없음
2단계 · 편집 ["Read","Glob","Grep","Edit","Write"] 파일 수정 가능 — 명령 실행 없음
3단계 · 실행 ["Bash"] + deny 차단 목록 권장 셸 명령 실행 자동 허용 — 신중히 사용
.claude/settings.json (2단계 편집 모드) 
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Edit", "Write"],
    "deny":  ["Bash(rm -rf *)"]
  }
}
세팅 5

서브에이전트 — 병렬 작업 · 컨텍스트 격리

고급

프롬프트에서 "서브에이전트로 병렬 처리"를 요청하면, 클로드 코드가 여러 AI를 동시에 실행하여 큰 작업을 빠르게 처리합니다. 컨텍스트 격리로 서로 간섭 없이 각 영역을 독립적으로 다룹니다.

활용 사례

대규모 감사

인증 누락, 보안 취약점을 엔드포인트별로 병렬 스캔

멀티 서비스 점검

마이크로서비스별 의존성 버전 충돌을 동시에 확인

동시 리팩토링

여러 모듈을 각각 독립적으로 수정 후 통합

프롬프트 예시 
services/ 아래 각 마이크로서비스의 package.json을 병렬로 스캔해줘.
버전 충돌, 순환 의존성, 취약점이 있는 패키지를 목록화.
서브에이전트로 각 서비스를 동시에 처리
세팅 6

MCP 서버 — 외부 데이터·도구 추가

고급

settings.json의 mcpServers에 외부 서버를 연결하면 사내 DB, 디자인 시스템, Slack, GitHub 등을 AI가 직접 읽고 쓸 수 있습니다.

자주 쓰는 MCP 서버

서버설치 명령용도
GitHub @modelcontextprotocol/server-github PR, 이슈, 저장소 관리
PostgreSQL @modelcontextprotocol/server-postgres DB 쿼리, 스키마 조회
Filesystem @modelcontextprotocol/server-filesystem 로컬 파일 시스템 접근
.claude/settings.json (GitHub + PostgreSQL) 
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_여기에_토큰_입력" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://user:password@localhost/mydb"]
    }
  }
}
