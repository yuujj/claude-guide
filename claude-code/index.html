핵심 개념

AI가 직접 실행합니다

클로드 코드는 "코딩 도구"라기보다, AI에게 작업을 직접 맡기는 도구입니다. 코딩을 몰라도 "이 폴더의 CSV 100개를 하나의 엑셀로 합쳐줘"처럼 자연어로 지시하면 됩니다. CLI 설치가 어렵다면 데스크탑 Code 탭(GUI)이나 claude.ai/code(웹)에서 쓸 수 있습니다.

📁

파일·데이터 정리

스크린샷 정리, CSV 합치기, 보고서 자동 생성 — 코딩 없이도 쓸 수 있습니다.

⚙️

개발 자동화

코드 작성, 리뷰, 테스트, PR 작성 — 개발자가 반복하던 작업을 맡깁니다.

🖥️

터미널 없이도 OK

데스크탑 Code 탭이나 웹(claude.ai/code)은 GUI로 쓸 수 있습니다.

시작

시작하기

5가지 환경에서 쓸 수 있습니다. 터미널이 낯설다면 데스크탑 Code 탭이나 웹부터 시작해보세요.

GUI · 터미널 불필요 Pro 이상
1
Claude Desktop 설치

Anthropic 공식 사이트에서 macOS 또는 Windows 설치 파일을 내려받아 실행합니다.

2
Code 탭 클릭

앱 상단에서 Code 탭을 선택합니다. 프로젝트 폴더를 열거나 끌어다 놓으면 됩니다.

3
자연어로 지시하기

채팅창에 원하는 작업을 입력합니다. AI가 직접 파일을 열고 수정합니다.

설치 불필요 Pro 이상
1
claude.ai/code 접속

브라우저에서 claude.ai/code를 열면 바로 클로드 코드 인터페이스가 나타납니다.

2
폴더 또는 파일 열기

로컬 파일을 업로드하거나 GitHub 저장소를 연결합니다.

3
지시하기

원하는 작업을 자연어로 입력하면 AI가 처리합니다.

VS Code 확장 Pro 이상
1
확장 설치

VS Code 확장 마켓에서 "Claude Code"를 검색하여 설치합니다.

2
Anthropic 계정 연결

확장 패널에서 로그인하면 편집기 내에서 바로 사용할 수 있습니다.

3
채팅창에서 지시하기

사이드바 채팅에서 코드 편집, 설명, 리뷰를 요청합니다. 현재 열린 파일 맥락을 자동으로 포함합니다.

JetBrains 플러그인 Pro 이상
1
플러그인 설치

JetBrains IDE의 플러그인 마켓에서 "Claude Code"를 검색하여 설치합니다.

2
계정 연결

설정 → Tools → Claude Code에서 Anthropic 계정으로 로그인합니다.

3
바로 사용

도구 창에서 작업을 지시하거나, 코드를 선택한 뒤 우클릭 → "Claude에 물어보기"를 선택합니다.

개발자 권장 Pro 이상
1
Node.js 설치 확인

Node.js 18 이상이 필요합니다. 아래 명령으로 버전을 확인합니다.

bash 
node --version
2
Claude Code 설치

공식 설치 페이지에서 네이티브 인스톨러를 사용하는 것을 권장합니다. npm으로 설치할 수도 있습니다.

bash (npm 방식) 
npm install -g @anthropic-ai/claude-code
3
프로젝트 폴더에서 실행
bash 
cd my-project
claude

처음 실행 시 Anthropic 계정 로그인 절차가 진행됩니다.

기능

핵심 기능

클로드 코드의 주요 기능입니다. 세팅에 관련된 고급 기능은 세팅 섹션에서 더 자세히 다룹니다.

AI가 지시에 따라 파일을 직접 열고, 수정하고, 저장합니다. 변경 전 diff를 보여주므로 어디가 바뀌었는지 확인한 뒤 승인할 수 있습니다.

셸 명령, 테스트 실행, 패키지 설치 등을 AI가 직접 수행합니다. 권한 모드 설정으로 자동 실행 범위를 조절할 수 있습니다.

스테이징, 커밋, 브랜치 생성, PR 초안 작성까지 AI가 처리합니다. 커밋 메시지 규칙은 CLAUDE.md에 정의해두면 자동으로 적용됩니다.

대규모 코드베이스도 AI가 구조를 파악하고 관련 파일을 찾아냅니다. "이 버그가 어디서 발생하는지 찾아줘"처럼 자연어로 요청하면 됩니다.

여러 서브에이전트가 코드베이스의 각 영역을 동시에 작업합니다. 컨텍스트 격리로 충돌 없이 병렬 진행이 가능하여, 대규모 감사나 리팩토링 작업을 크게 단축합니다.

비교

Cursor · Copilot과의 차이

기준: 2026년 4월 — 제품 업데이트에 따라 달라질 수 있습니다.

항목 클로드 코드 Cursor GitHub Copilot
방식 AI가 직접 작업 실행 AI 보조 편집 (IDE 내) AI 자동완성 · Chat
환경 터미널 · IDE · Desktop · 웹 Cursor IDE 전용 GitHub · 여러 IDE
파일 자율 편집 O (승인 후) O (Composer) 제한적
비개발자 사용 O (GUI 환경) X (IDE 기반) X (코딩 중심)
하네스 설정 CLAUDE.md · hooks · MCP Cursor Rules 제한적
무료 사용 X (Pro 이상) O (제한적 무료) O (개인 무료)
팁

팁

AI가 파일을 수정하기 전에 어떻게 분석하는지 먼저 지켜보세요. 권한 모드를 읽기 전용으로 설정하면 실수로 파일이 바뀌는 일 없이 탐색 단계를 진행할 수 있습니다.

삭제하면 안 되는 파일, 건드리면 안 되는 설정, 커밋 시 제외할 폴더 등을 CLAUDE.md에 명시해두면 실수를 크게 줄일 수 있습니다.

"전체를 리팩토링해줘" 보다 "우선 의존성 정리만 해줘. 그 다음에 모듈 분리를 하자"처럼 단계를 나누면 결과를 확인하면서 안전하게 진행할 수 있습니다.

같은 프롬프트를 두 번 이상 쓰고 있다면 슬래시 커맨드로 등록할 때입니다. .claude/commands/ 폴더에 마크다운 파일을 만들면 됩니다.
