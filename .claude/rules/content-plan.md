## 설계 원칙

1. **허브는 제품 기준, 페이지 안은 작업 기준** — "뭘 써야 하지?"에 즉답 가능하면서, 각 페이지 안에서는 "뭘 할 수 있지?"로 풀어 설명 (ADR-002)
2. **점층적 깊이** — 3문장 → disclosure → 하위 페이지
3. **도구는 선택지, 벽이 아님** — 클로드 코드를 "개발자 전용"으로 닫지 않음
4. **이진 분류 없음** — "비개발자/개발자" 선택을 강요하지 않음. 적응형 컴포넌트가 자연스럽게 대응 (ADR-001)

## 클로드 코드 포지셔닝 노트

클로드 코드는 본질적으로 "AI가 직접 행동하는 도구"이지 "코딩 도구"가 아님.
- 파일을 읽고 수정하고, 명령을 실행하고, git을 다루는 건 개발 맥락에서 가장 많이 쓰이지만
- "이 폴더 CSV 100개를 정리해줘", "매주 보고서 자동 생성해줘" 같은 작업도 가능
- 현실적으로 CLI 설치+터미널 사용이 진입장벽 -> Desktop Code 탭, claude.ai/code가 대안

**가이드에서의 접근법**:
- `claude-code/` 페이지 상단에 "이건 코딩 도구인가?" 오해 해소 섹션
- 비개발자 예시(파일 정리, CSV 합치기, 보고서 자동 생성)를 개발자 예시보다 먼저 배치
- 시작 방법에서 Desktop Code 탭(GUI)을 CLI보다 먼저 안내

## 사이트 전체 구조

```
index.html                    <- 허브: 클로드 소개 + 통합 제품 카드 3장 (유스케이스/기능/접근성 3렌즈 병합)
claude-web/index.html         <- 클로드 웹 (claude.ai)
claude-desktop/index.html     <- 클로드 데스크탑 (Chat/Code/Cowork 3탭)
claude-code/index.html        <- 클로드 코드 (CLI/IDE/Desktop/Web)
tip/index.html           <- TIP (ADR-001)
```

### 세 제품의 핵심 차이

| | 클로드 웹 | 클로드 데스크탑 | 클로드 코드 |
|---|---|---|---|
| **환경** | 브라우저 (claude.ai) | macOS / Windows 앱 | 터미널 / IDE / Desktop 탭 / 웹 |
| **진입 난이도** | 가장 쉬움 | 보통 | 보통 (GUI) ~ 높음 (CLI) |
| **포함 기능** | 채팅, 프로젝트, 아티팩트, 웹 검색, 메모리 | 웹의 모든 기능 + MCP + Code + Cowork | 파일 편집, 명령 실행, git, 서브에이전트 |
| **핵심 차별점** | 설치 없이 즉시 시작 | 내 환경과 연결 (MCP), GUI로 Code 쓰기 | AI가 직접 작업 실행 |
| **무료 사용** | O | O (기본 Chat) / Code·Cowork은 Pro 이상 | X (Pro 이상) |

### 제품 간 포함 관계 (중요)

```
클로드 웹 ⊂ 클로드 데스크탑
              └─ 데스크탑 Code 탭 ≈ 클로드 코드 (GUI 래퍼)
              └─ 데스크탑 Cowork 탭 (데스크탑 고유)
```

데스크탑 페이지에서는 "웹 기능은 여기서 다 됩니다"로 가볍게 링크, 고유 기능(MCP, Cowork, Code탭 GUI)에 집중.

## 청중 대응 전략 (ADR-001)

### 원칙: 이진 선택 없음 — 컴포넌트가 적응

청중 선택 카드를 **두지 않는다**. 대신 3가지 적응형 컴포넌트가 각 청중의 핵심 문제를 해결:

| 컴포넌트 | 비개발자 문제 해결 | 개발자 문제 해결 |
|---------|------------------|-----------------|
| **용어 툴팁** | 기술 용어 → 탭하면 정의 표시 | (자연스럽게 무시) |
| **사이드바 ToC + 배지** | `[고급]` 보면 건너뛰어도 됨 인식 | `[고급]` 이 비콘 → 바로 점프 |
| **TIP 페이지** | 일러스트로 개념 이해 | 빠른 참조 + 정확한 정의 |

### 용어 툴팁 사양

```html
<span class="term" data-def="AI가 외부 도구와 통신하는 개방형 표준 규격"
      data-tip="mcp">MCP</span>
```

- **스타일**: `border-bottom: 1.5px dotted var(--color-accent)`, `cursor: help`
- **동작**: 클릭/호버 -> 팝오버 (정의 1줄 + "TIP에서 자세히" 링크)
- **모바일**: 탭 -> 표시, 바깥 탭 -> 닫기
- **접근성**: `aria-describedby` 연결
- **규칙**: 페이지당 각 용어의 **첫 등장**에만 적용. 문단당 최대 1개

### 사이드바 ToC 사양

- **데스크탑**: sticky 사이드바, IntersectionObserver로 현재 섹션 하이라이트
- **모바일**: sticky 드롭다운 ("이 페이지: [현재 섹션명]")
- **배지**: 고급 섹션에 `<span class="badge badge--dev">고급</span>` 표시
- **위치**: 콘텐츠 왼쪽 (허브 제외, 제품 페이지에만 적용)

### disclosure 배지

기존 disclosure 트리거에 깊이 표시 배지 인라인:
```
더 알아보기 — MCP 직접 설정 [고급]
더 알아보기 — 모델 패밀리 [기초]
```

## TIP 페이지 (tip/index.html)

### 목적
기술 용어를 비개발자도 이해할 수 있게 설명하는 독립 참조 페이지.
용어 툴팁의 "자세히 보기" 링크가 이 페이지의 해당 항목으로 연결.

### 구조
```
[NAV]
[Hero]: "TIP — 클로드 용어를 쉽게 이해하세요"
[검색/필터]: 텍스트 입력으로 실시간 필터링
[카테고리 탭]: 전체 | 기본 | 제품 기능 | 연결 | 자동화
[용어 목록]: 카드형 그리드
[Footer]
```

### 각 용어 항목 구조
```
[아이콘/일러스트]
[용어명]  (영문 원어)
[한 줄 정의]
[비유] — "USB처럼 꽂으면 연결되는..."
[disclosure: 기술 상세]
[관련 페이지 링크]
```

## 클로드 웹 페이지 (claude-web/index.html)

### 한 줄 정의
브라우저에서 바로 시작하는 AI — 설치 없이 대화하고, 분석하고, 만들 수 있습니다.

### 섹션 구조
```
[1] Hero + CTA [claude.ai 열기]
[2] 시작하기 (3단계)
[3] 이런 걸 할 수 있어요 (작업별 서브섹션)
    - 질문·학습
    - 글쓰기·번역·요약
    - 분석 (이미지·데이터)
    - 만들기 (아티팩트)
[4] 핵심 기능 (아코디언: 프로젝트, 아티팩트, 메모리, 웹 검색, 커스텀 스타일)
[5] 활용 예시 (시나리오 카드)
[6] 요금·한계 (간단히 + 공식 링크)
[7] 팁
[8] 다음 단계 -> 데스크탑 / 코드
```

### [3] 이런 걸 할 수 있어요

**컴포넌트**: `example-card` 그리드 (auto-fit, min 280px)
각 카드 = 이모지 아이콘 + 제목 + 짧은 설명 + 코랄 좌측 보더의 이탤릭 프롬프트 인용.
프롬프트는 **사용자가 복붙해서 바로 써볼 수 있는 수준**으로 구체적이어야 함.

| 아이콘 | 작업 카테고리 | 설명 (한 줄) | 예시 프롬프트 |
|--------|--------------|--------------|--------------|
| 💬 | 질문·학습 | 어려운 개념을 내 수준에 맞춰 설명받고, 후속 질문으로 깊이 들어갑니다. | "양자역학을 중학생 수준으로 설명하고, 이해도 확인할 퀴즈 3개 만들어줘" |
| ✍️ | 글쓰기 | 상황과 톤을 주면 초안을 만들어줍니다. 커스텀 스타일로 회사 톤을 고정할 수도 있습니다. | "거래처에 납기 지연 안내 메일 — 정중하지만 단호하게, 3문단으로" |
| 🖼️ | 이미지·파일 분석 | PDF, 엑셀 스크린샷, 영수증 사진을 업로드하면 내용을 읽고 정리합니다. | "이 영수증 10장을 표로 정리하고 월별 지출 카테고리 분석" |
| ⚡ | 만들기 (아티팩트) | 채팅 옆에 실시간으로 프로토타입이 렌더링됩니다. | "연봉 협상 시뮬레이터 — 연봉/세금/실수령액 보여주는 인터랙티브 페이지" |

(추가 카테고리 — 번역·요약, 코드 스니펫 생성 등 — 은 `disclosure` 로 확장 가능)

## 클로드 데스크탑 페이지 (claude-desktop/index.html)

## 클로드 코드 페이지 (claude-code/index.html)

### 한 줄 정의
AI에게 작업을 맡기는 도구 — 파일을 직접 수정하고, 명령을 실행하고, 결과를 보여줍니다.

### 섹션 구조
```
[1] Hero + CTA
[2] 이건 코딩 도구인가? (비개발자 해명)
[3] 시작하기 (5가지 surface 탭: 데스크탑 Code탭 / 웹 / VS Code / JetBrains / 터미널)
[4] 이런 걸 맡길 수 있어요 (시나리오 리스트, 난이도 순)
[5] 핵심 기능 (아코디언, [고급] 배지 활용)
[6] 활용 예시 상세
[7] 비교: Cursor vs Copilot (기준일 표기)
[8] 반복 가능한 세팅으로 만들기 — 하네스 엔지니어링 [고급]
[9] 팁
[10] 다음 단계
```

### [2] 이건 코딩 도구인가?
```
클로드 코드는 "코딩 도구"라기보다, AI에게 작업을 직접 맡기는 도구입니다.
코딩을 몰라도, "이 폴더의 CSV 100개를 하나의 엑셀로 합쳐줘"처럼 자연어로 지시하면 됩니다.
단, CLI 설치가 어렵다면 데스크탑 Code 탭(GUI)이나 claude.ai/code(웹)에서 쓸 수 있습니다.
```

### [4] 이런 걸 맡길 수 있어요

**컴포넌트**: `scenario` 리스트 — 난이도 점(🟢🟡🔵) 좌측 레일 + 제목 옆 배지(`비개발자`/`개발자`/`하네스`) + 터미널 풍 코드블록 프롬프트.

**원칙**:
- 난이도 순으로 나열 (기초 → 고급). 비개발자가 먼저 눈에 들어오도록 함
- 프롬프트는 실제 작동할 수준으로 구체적으로 — 파일명, 폴더 구조, 규칙을 포함
- `하네스` 배지는 "단발 프롬프트 → 반복 가능한 세팅" 의 진입점. 섹션 [8]로 연결되는 시각적 훅
- `웹` 예시는 따옴표 인용(자연어), `코드` 예시는 터미널 코드블록 — 시각적 구분으로 "사람이 읽는 것 vs AI가 읽는 것" 암시

| 난이도 | 제목 | 배지 | 상황 | 예시 프롬프트 |
|--------|------|------|------|--------------|
| 🟢 기초 | 스크린샷 폴더 자동 정리 | `비개발자` | Desktop에 쌓인 스크린샷 수백 장을 월별 서브폴더로 나누고 파일명 규칙화. | `이 폴더의 .png 파일 모두, 생성일 기준으로 YYYY-MM 서브폴더에 옮기고, 파일명은 YYYY-MM-DD_원래이름.png 로 바꿔줘` |
| 🟢 기초 | CSV 100개 합치기 | `비개발자` | 여러 부서에서 받은 지점별 매출 CSV를 하나로 통합·중복 제거. | `이 폴더의 모든 csv를 하나로 합쳐서 total.xlsx 로 저장. 컬럼 이름 통일하고 지점명 기준 중복 제거해줘` |
| 🟡 중간 | 주간 보고서 자동화 | `비개발자` `하네스` | 매주 월요일 같은 작업 반복 → 슬래시 커맨드 한 번으로. | `/weekly-report` (커맨드 정의 — 이 폴더 데이터 읽고, 템플릿 채우고, PDF로 저장) |
| 🔵 고급 | 실패 테스트 원인 찾아 수정 | `개발자` | CI에서 빨간 테스트의 원인 파악 + 회귀 테스트 추가 + PR 초안. | `CI 로그 보고 실패 원인 찾아서 수정. 회귀 테스트 추가하고 PR 초안 작성해줘 — 리뷰어가 이해할 수 있게 맥락 포함` |
| 🔵 고급 | API 인증 검사 누락 감사 | `개발자` `하네스` | 서브에이전트 병렬 처리로 엔드포인트별 인증 검사 훑기. | `routes/ 아래 엔드포인트 전부 스캔. 인증 미들웨어 누락된 것, 권한 검사 없이 데이터 반환하는 것 목록화. 서브에이전트로 병렬 처리` |

### [8] 반복 가능한 세팅으로 만들기 — 하네스 엔지니어링 [고급]

> 목적: 단발 프롬프트에서 **"한 번 세팅하면 계속 쓰는 도구"** 로 넘어가는 다리.
> [4] 에서 `하네스` 배지가 붙은 시나리오(주간 보고서 / API 감사) 의 뒷이야기.

**하네스란?** — LLM을 감싸는 스캐폴딩. 같은 모델도 하네스에 따라 생산성이 10배 차이. (TIP 페이지 `하네스` 용어 카드와 연결)

**구성 요소** (각 항목은 disclosure로 확장, 실전 예시 포함):

1. **CLAUDE.md** — 프로젝트 규칙 자동 주입 (매 세션 자동 로드)
2. **슬래시 커맨드 / skills** — 자주 쓰는 프롬프트 표준화 (`/weekly-report`)
3. **hooks** — 특정 이벤트에 자동 실행 (커밋 전 린트, 세션 종료 시 요약)
4. **권한 모드 / settings.json** — 안전하게 자율성 늘리기 (read-only → edit → execute)
5. **서브에이전트** — 병렬 작업 / 컨텍스트 격리 (대규모 감사, 여러 영역 동시 수정)
6. **MCP 서버** — 외부 데이터·도구 추가 (사내 DB, 디자인 시스템, 업무 툴 연동)

**톤**: "이걸 알면 좋지만, 몰라도 기초 예시는 다 쓸 수 있습니다" — [고급] 배지로 명확히 구분.

## 콘텐츠 톤 가이드

| 항목 | 규칙 |
|------|------|
| 문체 | 경어체 (~합니다) — 친근하되 가벼워지지 않게 |
| 전문 용어 | 첫 등장 시 `<span class="term">` 으로 마크업. 정의는 `data-def` 속성에 |
| 비유 | 기술 개념에 일상 비유 적극 사용 |
| 코드 블록 | 비개발자 페이지에서는 최소한. 자연어 예시 우선 |
| 길이 | 기본 3문장 이내 -> disclosure로 확장 |
| 진입장벽 | "코딩을 몰라도", "설치 없이", "클릭 몇 번으로" 접근성 강조 |
| 도구 강요 | "꼭 이것만" X -> "이런 상황이면 이게 편합니다" O |
| 깊이 표시 | disclosure 트리거에 `[고급]` / `[기초]` 배지 인라인 |
| 공식 문서 | 각 제품 페이지 최하단에 "공식 문서 →" 링크 일관 배치 |

### 종결어미 규칙 (혼재 금지)

위치 종류별로 종결 형태가 다름. **같은 섹션 안에서 섞지 않음**.
특히 "...할 때. ...지금 바로." 같은 중간형(부사절/명사구 + 마침표) 은 **금지**.

| 위치 종류 | 종결 형태 | 예 (O) | 예 (X) |
|----------|---------|-------|-------|
| 본문 / 카드 설명 문장 | 완전한 합니다체, 마침표 찍음 | "브라우저만 있으면 됩니다." | "브라우저만 있으면 됨." |
| 부제 / 메타 / 칩 / 키워드 | 구(句) 형태, 마침표 없음, `·` 로 연결 | "claude.ai · 브라우저에서 바로" | "claude.ai에서 바로 씁니다." (부제에 문장 금지) |
| 시나리오 상황 설명 | 완전한 합니다체 1문장 | "스크린샷 수백 장을 월별 서브폴더로 정리합니다." | "수백 장을 월별로 정리." |
| 프롬프트 예시 | **사용자 말투 그대로** (명령형/반말 가능), 따옴표·코드블록 인용 | "이 폴더 CSV 합쳐줘" | "이 폴더의 CSV를 합쳐 주시기 바랍니다." (실제 안 쓰는 말투) |
| CTA 버튼 | 명령형 or 명사 한 단어 | "시작하기", "자세히 보기" | "시작해 봐요 →" |
| 섹션 라벨 (`section-label`) | 한 단어 명사 | "제품", "가이드", "활용" | "우리의 제품들" |

### 용어 마크업 규칙

```html
<!-- 올바른 예시: 첫 등장, 정의 포함 -->
<span class="term" data-def="AI가 외부 도구와 통신하는 표준 규격"
      data-tip="mcp">MCP</span>를 통해 연결합니다.

<!-- 이후 등장: 일반 텍스트 -->
MCP 서버를 설치하면...
```

- 페이지당 각 용어의 **첫 등장에만** `<span class="term">` 적용
- 같은 문단에 2개 이상의 term을 넣지 않음
- `data-tip` 값은 TIP 페이지의 앵커 ID와 일치
