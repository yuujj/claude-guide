## 클로드 코드 포지셔닝 노트

클로드 코드는 본질적으로 "AI가 직접 행동하는 도구"이지 "코딩 도구"가 아님.
- 파일을 읽고 수정하고, 명령을 실행하고, git을 다루는 건 개발 맥락에서 가장 많이 쓰이지만
- "이 폴더 CSV 100개를 정리해줘", "매주 보고서 자동 생성해줘" 같은 작업도 가능
- 단, 현실적으로 CLI 설치·터미널 사용이 진입장벽 → Desktop Code 탭, claude.ai/code가 대안

**가이드에서의 접근법**:
- "맡기기" 페이지에서 사용 사례를 **개발자 한정이 아닌 "AI에게 직접 작업을 위임"**으로 프레이밍
- 개발자 예시와 비개발자 예시를 나란히 제시하되, 비개발자 예시가 먼저
- 설치 방법에서 CLI뿐 아니라 Desktop Code 탭, 웹 버전을 동등하게 안내
- "코딩을 몰라도 됩니다. 자연어로 지시하면 AI가 실행합니다."라는 톤

## 사이트 전체 구조

```
index.html          <- 허브: 클로드 소개 + "뭘 하고 싶으세요?" 작업 분기
chat/index.html     <- 대화하기: AI에게 질문하고, 글·분석·번역 요청
connect/index.html  <- 연결하기: 내 파일, 앱, 데이터를 AI와 연결
automate/index.html <- 맡기기: AI가 직접 파일 수정, 명령 실행, 작업 완료
```

### 세 페이지의 핵심 차이

| | 대화하기 | 연결하기 | 맡기기 |
|---|---|---|---|
| **AI가 하는 일** | 답변·생성 | 내 데이터를 읽고 답변 | 직접 파일 수정·명령 실행 |
| **비유** | 전문가에게 상담 | 비서에게 자료 접근권 부여 | 대리인에게 업무 위임 |
| **사용 도구** | 웹, 데스크탑 Chat | 데스크탑 MCP, 커넥터 | Code, Cowork |
| **진입 난이도** | 가장 쉬움 | 보통 | 보통~높음 |
| **무료 가능** | O | O (데스크탑) | 부분 (Cowork은 유료) |

## 대화하기 페이지 (chat/index.html)

### 한 줄 정의
AI에게 질문하고, 글을 쓰고, 분석을 맡기세요 — 브라우저만 있으면 됩니다.

### 사용 도구
- 클로드 웹 (claude.ai)
- 클로드 데스크탑 Chat 탭 (동일한 기능)
- 모바일 앱 (iOS / Android)

### 섹션 구조

```
[1] Hero: "AI에게 물어보세요" + CTA [claude.ai 열기]
[2] 시작하기: 3단계 (접속 -> 질문 -> 프로젝트 정리)
[3] 이런 걸 할 수 있어요 (작업별 카드 그리드)
[4] 핵심 기능 (아코디언)
[5] 활용 예시 (구체적 시나리오)
[6] 팁
[7] 다음 단계: "더 하고 싶다면" -> 연결하기 / 맡기기 링크
```

### [3] 이런 걸 할 수 있어요

| 작업 | 설명 | 예시 프롬프트 |
|------|------|--------------|
| 글쓰기 | 이메일, 보고서, 블로그 초안 | "거래처에 납기 지연 안내 메일 써줘" |
| 요약 | 긴 문서, 회의록, 논문 핵심 추출 | "이 PDF 3줄로 요약해줘" |
| 번역 | 자연스러운 한↔영 번역 | "이 영문 계약서를 한국어로 번역해줘" |
| 분석 | 데이터, 차트, 스크린샷 해석 | "이 엑셀 스크린샷에서 매출 트렌드 분석해줘" |
| 코드 도움 | 코드 설명, 디버깅, 생성 | "이 에러 메시지가 무슨 뜻이야?" |
| 아이디어 | 브레인스토밍, 구조 잡기 | "마케팅 캠페인 아이디어 5개 제안해줘" |
| 학습 | 개념 설명, 퀴즈, 튜터링 | "양자역학을 중학생도 이해하게 설명해줘" |

### [4] 핵심 기능

#### 프로젝트
- 관련 대화·파일·지침을 한 곳에 묶는 작업 공간
- 파일을 한 번 올리면 해당 프로젝트의 모든 대화에서 참조
- 팀과 공유 가능 (Team/Enterprise)
- 예시: "브랜드 가이드" 프로젝트 -> 모든 마케팅 대화에서 톤 유지

#### 아티팩트
- 클로드가 만든 코드·문서·다이어그램을 채팅 옆에 실시간 렌더링
- 반복 수정: "색상 바꿔줘", "차트 형식 변경"
- 공유 링크로 다른 사람에게 보내기
- 지원: HTML/CSS/JS, React, SVG, Mermaid 다이어그램, 계산기, 게임

#### 메모리
- 이전 대화를 기억하여 다음 세션에 반영
- 프로젝트별 별도 메모리
- 무료 플랜에서도 사용 가능

#### 웹 검색
- 실시간 인터넷 검색으로 최신 정보 반영
- 출처 링크 함께 제공

#### 커스텀 스타일
- 응답 톤 개인화: "간결하게", "친근하게", "전문적으로"
- 설정 한 번이면 모든 대화에 적용

### [5] 활용 예시

| 시나리오 | 상세 |
|---------|------|
| 주간 보고서 | 프로젝트에 양식 저장 -> 매주 데이터 붙여넣기 -> "이번 주 보고서 작성해줘" |
| 코드 프로토타입 | "React로 투두앱 만들어줘" -> 아티팩트에서 실시간 확인 -> 공유 링크 전달 |
| 논문 리뷰 | PDF 업로드 -> "방법론의 약점을 지적해줘" -> 개선안 토론 |
| 다국어 대응 | 영문 마케팅 카피 -> "한국어, 일본어로 현지화해줘" |

### [6] 팁
- 프로젝트 "커스텀 지침"을 쓰면 매번 같은 말 반복 안 해도 됨
- 긴 문서는 한 번에 올리는 게 나누어 올리는 것보다 정확도 높음
- 아티팩트 "Remix"로 다른 사람의 공유물을 내 것으로 수정 가능

## 연결하기 페이지 (connect/index.html)

### 한 줄 정의
내 컴퓨터의 파일, 데이터베이스, 업무 도구를 AI에게 열어주세요 — AI가 직접 참조합니다.

### 사용 도구
- 클로드 데스크탑 (Chat 탭 + MCP)
- 클로드 웹/데스크탑 커넥터 (Slack, Google Drive 등)
- Claude Code의 MCP (개발자 설정)

### 핵심 개념: "대화하기"와의 차이

```
대화하기: 사용자가 파일을 업로드 -> AI가 읽음 -> 답변
연결하기: AI가 직접 파일/DB/앱에 접근 -> 필요한 것을 스스로 찾아서 답변
```

### 섹션 구조

```
[1] Hero: "내 환경을 AI와 연결하세요" + CTA [데스크탑 다운로드]
[2] 왜 연결이 필요한가? (대화하기와 비교)
[3] 연결 방법 3가지 (커넥터 / MCP GUI / MCP CLI)
[4] MCP란? (쉬운 설명 + disclosure로 기술 상세)
[5] 활용 예시
[6] 시작하기 (난이도별 탭)
[7] 다음 단계 -> 맡기기
```

### [2] 왜 연결이 필요한가?

| | 대화하기 (업로드) | 연결하기 (MCP/커넥터) |
|---|---|---|
| 데이터 전달 | 매번 파일을 직접 올림 | AI가 필요할 때 스스로 접근 |
| 최신성 | 올린 시점의 데이터 | 항상 최신 |
| 범위 | 올린 파일만 | 폴더 전체, DB, 앱 데이터 |
| 예시 | "이 CSV 분석해줘" (파일 드래그) | "이번 달 매출 데이터 분석해줘" (AI가 직접 DB 조회) |

### [3] 연결 방법 3가지

#### 방법 1: 커넥터 (가장 쉬움)
- 클로드 웹/데스크탑에서 기본 제공
- Slack, Google Drive, Google Workspace, Notion 등
- 설정: 클릭 몇 번으로 연결
- Team/Enterprise 플랜 필요

#### 방법 2: 데스크탑 확장 (쉬움)
- `.mcpb` 파일 더블클릭 -> 설치 완료
- 파일시스템, Git, 각종 서비스
- 자격증명은 OS 키체인에 암호화 저장
- 데스크탑 앱 필요

#### 방법 3: MCP 직접 설정 (개발자)
- CLI에서 `claude mcp add` 또는 JSON 직접 편집
- 프로젝트/사용자/로컬 3단계 스코프
- 팀 공유 가능 (`.mcp.json` 버전관리)
- 가장 유연하지만 터미널 필요

### [4] MCP란?

**기본 노출**:
```
MCP는 AI가 내 컴퓨터의 프로그램과 대화할 수 있게 해주는 표준 규격입니다.
USB처럼 — 장치를 꽂으면 컴퓨터가 자동으로 인식하듯,
MCP 서버를 설치하면 클로드가 해당 서비스를 자동으로 사용할 수 있습니다.
```

**disclosure 확장**: 프로토콜 구조, stdio/HTTP 전송, 스코프 시스템, 인기 MCP 서버 목록

### [5] 활용 예시

| 시나리오 | 연결 대상 | 예시 |
|---------|----------|------|
| 로컬 파일 분석 | 파일시스템 MCP | "Documents 폴더에서 이번 달 매출 보고서 찾아서 분석해줘" |
| DB 조회 | PostgreSQL MCP | "최근 7일 가입자 수 알려줘" (SQL 자동 생성) |
| Slack 요약 | Slack 커넥터 | "이번 주 #design 채널 논의 요약해줘" |
| Google Drive | Drive 커넥터 | "공유 드라이브에서 마케팅 기획서 찾아서 분석해줘" |
| Figma 참조 | Figma MCP | 디자인 파일 참조하며 CSS 코드 생성 |

### [6] 시작하기

**탭: 커넥터 (가장 쉬움) | 데스크탑 확장 | CLI 설정**

## 맡기기 페이지 (automate/index.html)

### 한 줄 정의
AI가 직접 파일을 수정하고, 명령을 실행하고, 작업을 완료합니다 — 말로 지시하면 됩니다.

### 사용 도구
- Claude Code (Desktop Code 탭 / CLI / VS Code / JetBrains / 웹)
- Cowork (Desktop Cowork 탭)
- 스케줄 작업

### 핵심 개념: "대화하기", "연결하기"와의 차이

```
대화하기: AI가 "답변"을 줌     -> 사용자가 직접 실행
연결하기: AI가 데이터를 "읽음"  -> 사용자가 직접 실행
맡기기:   AI가 직접 "실행"함   -> 사용자는 결과만 확인
```

### 섹션 구조

```
[1] Hero: "AI에게 작업을 맡기세요" + CTA
[2] "맡기기"란? (대화/연결과 비교)
[3] 이런 걸 맡길 수 있어요 (작업 카드 — 비개발자 예시 먼저)
[4] 두 가지 방식: Code vs Cowork
[5] 시작하기 (난이도별 탭)
[6] 핵심 기능 (아코디언)
[7] 활용 예시 상세
[8] 팁
[9] 다음 단계
```

### [2] "맡기기"란?

```
지금까지는 AI에게 "물어보고" 답을 받거나, 데이터를 "연결해서" 참조하게 했습니다.
"맡기기"는 한 단계 더 — AI가 직접 파일을 수정하고, 프로그램을 실행합니다.

코딩을 몰라도 됩니다.
"이 폴더의 파일 이름을 날짜순으로 바꿔줘"처럼 자연어로 지시하면,
AI가 알아서 실행하고 결과를 보여줍니다.
```

### [3] 이런 걸 맡길 수 있어요

**비개발자 예시 (먼저 노출)**:

| 작업 | 설명 |
|------|------|
| 파일 정리 | "이 폴더의 이미지를 날짜별로 분류해줘" |
| 데이터 변환 | "이 CSV 100개를 하나의 엑셀로 합쳐줘" |
| 보고서 자동 생성 | "매주 월요일에 이 데이터로 보고서 만들어줘" |
| 문서 일괄 수정 | "모든 마크다운 파일에서 회사명을 변경해줘" |
| 브라우저 작업 (Cowork) | "이 웹사이트에서 데이터를 추출해서 스프레드시트에 넣어줘" |

**개발자 예시**:

| 작업 | 설명 |
|------|------|
| 코드 수정 | "auth 모듈을 OAuth2로 마이그레이션해줘" |
| 테스트 작성+실행 | "이 모듈 테스트 작성하고 통과시켜줘" |
| git 관리 | "변경사항 커밋하고 PR 만들어줘" |
| 리팩토링 | "이 파일 함수를 클래스로 변환해줘" |
| CI/CD 통합 | "PR마다 자동으로 보안 검토 실행해줘" |

### [4] 두 가지 방식

#### Claude Code — 파일과 명령을 직접 다루는 AI

```
자연어로 지시 -> AI가 파일 읽기/수정, 셸 명령 실행, git 관리
결과를 확인 -> 승인하면 적용
```

- 어디서든: 데스크탑 Code 탭 / 터미널 CLI / VS Code / JetBrains / 웹
- 사용자 승인 없이는 아무것도 바꾸지 않음 — 항상 확인 후 적용
- CLAUDE.md로 프로젝트 규칙 저장 -> 모든 세션에서 일관된 작업

#### Cowork — 화면을 보고 앱을 직접 조작하는 AI

```
작업 지시 -> AI가 마우스·키보드로 앱을 조작 -> 결과 확인
```

- 데스크탑 Cowork 탭에서 사용
- 브라우저, 스프레드시트, 사내 앱 등 GUI가 있는 모든 앱
- 기술 지식 불필요 — 사람이 하는 것처럼 화면을 보고 동작

#### 언제 뭘 쓸까?

| 상황 | Claude Code | Cowork |
|------|:-:|:-:|
| 파일 수정·정리 | O (빠름) | O (느림) |
| 코드 편집 + git | O (최적) | X |
| 웹 브라우저 조작 | X | O (최적) |
| GUI 앱 (엑셀, Figma 등) | X | O (최적) |
| 배치 처리 (100개 파일) | O (최적) | X |
| 반복 스케줄 | O | X |

### [5] 시작하기

**탭: 데스크탑 앱 (가장 쉬움) | 웹에서 바로 | 터미널 CLI**

탭 A: 데스크탑 앱
```
1. 클로드 데스크탑 설치 (이미 있다면 건너뛰기)
2. Code 탭 또는 Cowork 탭 선택
3. 작업할 폴더를 열고 자연어로 지시
```

탭 B: 웹에서 바로
```
1. claude.ai/code 접속
2. GitHub 저장소 연결
3. 자연어로 작업 지시 (클라우드 환경에서 실행)
```

탭 C: 터미널 CLI
```
curl -fsSL https://claude.ai/install.sh | bash
cd my-project && claude
"이 폴더의 CSV 파일을 모두 합쳐줘"
```

### [6] 핵심 기능 (Claude Code)

#### 파일 읽기/수정
- 프로젝트 전체 파일을 읽고 맥락을 파악
- 여러 파일을 동시에 수정 (의존성 인식)
- 변경 전 반드시 diff로 확인

#### 명령 실행
- 셸 명령, 빌드, 테스트, 린터
- 실패 시 자동 분석 + 수정 제안

#### git 통합
- 커밋, 브랜치, PR 생성
- GitHub/GitLab 직접 연동

#### CLAUDE.md 프로젝트 메모리
- 프로젝트 규칙을 파일로 저장 -> 모든 세션에서 자동 적용
- 팀원 전체가 같은 컨텍스트 공유

#### 서브에이전트 [고급]
- 여러 AI가 다른 부분을 동시에 작업
- 대규모 리팩토링, 멀티모듈 변경에 유용

#### 스케줄 작업 [고급]
- 반복 작업 예약: "매일 아침 CI 실패 요약"
- 데스크탑 또는 클라우드에서 실행

### [7] 활용 예시 상세

| 시나리오 | 대상 | 상세 |
|---------|------|------|
| 파일 일괄 변환 | 비개발자 | 100개 CSV를 하나의 정리된 엑셀로 합치기 |
| 폴더 정리 자동화 | 비개발자 | 사진 파일을 촬영일 기준 폴더로 자동 분류 |
| 보고서 스케줄 | 비개발자 | 매주 월요일 데이터 파일에서 보고서 자동 생성 |
| 웹 데이터 수집 | 비개발자 | Cowork로 웹사이트에서 정보 추출 -> 스프레드시트 저장 |
| 테스트 자동화 | 개발자 | 모듈별 테스트 작성, 실행, 실패 수정까지 한 번에 |
| PR 리뷰 자동화 | 개발자 | `git diff main \| claude -p "보안 이슈 검토"` |
| 다국어 자동화 | 개발자 | CI에서 새 문자열 자동 번역 + PR 생성 |

### [8] 팁
- 처음이라면 Desktop Code 탭으로 시작 — CLI보다 시각적
- CLAUDE.md에 프로젝트 규칙 적어두면 정확도 크게 올라감
- Cowork는 사람이 하는 것처럼 화면을 "보고" 동작하므로, 복잡한 GUI 작업에 적합

## 청중 분기 전략 (v2)

### 변경점

v1에서는 "일반 사용자 / 개발자" 2분법 + 배지로 분기.
v2에서는 **작업 기준 구조 자체가 청중 분기 역할을 대체**:

- 대화하기 -> 모든 사용자가 자연스럽게 진입
- 연결하기 -> 로컬 데이터 활용에 관심 있는 사용자
- 맡기기 -> 자동화/개발 필요한 사용자

### 남은 분기 포인트

허브의 청중 선택 카드는 **제거** (작업 카드가 대체).
대신 각 페이지 내에서:
- disclosure로 기술 상세를 접어둠 (관심 있으면 펼침)
- 맡기기 페이지에서 비개발자 예시를 개발자 예시보다 먼저 배치
- 배지는 유지하되, "일반/개발자" 대신 "쉬움/고급" 느낌으로
